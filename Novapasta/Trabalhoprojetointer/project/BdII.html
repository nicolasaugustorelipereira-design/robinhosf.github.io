1º Bimestre

O MySQL é um dos sistemas de gerenciamento de banco de dados mais populares do mundo. Ele é gratuito, de código aberto e amplamente utilizado em aplicações web, desde pequenos sites até grandes plataformas online.

Sua principal característica é a utilização do modelo relacional, permitindo armazenar e gerenciar dados em tabelas. O MySQL é otimizado para velocidade e eficiência, o que o torna uma escolha comum para aplicações que precisam lidar com grandes volumes de informações.

Algumas das principais vantagens do MySQL são:

  • Gratuito – pode ser usado sem custos na maioria dos projetos;
  • Portabilidade – funciona em diferentes sistemas operacionais;
  • Escalabilidade – suporta desde pequenos sites até aplicações robustas;
  • Comunidade ativa – possui vasto material de estudo e suporte online.

O MySQL é frequentemente utilizado em conjunto com outras tecnologias, como o PHP e o Apache, formando a popular pilha LAMP (Linux, Apache, MySQL e PHP).

Exemplo de uso do MySQL em um sistema de e-commerce: ele armazena informações sobre produtos, clientes e pedidos, permitindo que os usuários façam buscas rápidas e que os administradores tenham total controle sobre o catálogo e as vendas.

O MySQL foi desenvolvido em 1995 por Michael Widenius (conhecido como “Monty”) e David Axmark, na empresa sueca MySQL AB. O objetivo era criar um sistema de banco de dados rápido, eficiente e de código aberto que pudesse ser utilizado em aplicações reais de alto desempenho.

Seu nome vem da combinação de “My”, que é o nome da filha de Widenius, e SQL, a linguagem padrão de consulta a bancos de dados relacionais.

Durante sua evolução, o MySQL conquistou grande popularidade, tornando-se parte da pilha LAMP (Linux, Apache, MySQL, PHP), amplamente usada para o desenvolvimento de aplicações web.

A história do MySQL pode ser dividida em etapas:

  • 1995 – Lançamento oficial da primeira versão pela MySQL AB;
  • 2008 – A empresa Sun Microsystems adquire a MySQL AB;
  • 2010 – A Oracle Corporation compra a Sun e assume o desenvolvimento do MySQL;
  • Atualmente – O MySQL continua sendo mantido pela Oracle, com versões comerciais e gratuitas.

Com o passar do tempo, o MySQL se consolidou como um dos sistemas de gerenciamento de banco de dados mais importantes do mundo, utilizado por gigantes como Facebook, YouTube, Twitter e WordPress.

Na época, muitas soluções existentes eram caríssimas ou possuíam licenciamento fechado, dificultando o uso por desenvolvedores independentes e empresas menores.

A principal razão do desenvolvimento foi oferecer um banco de dados de código aberto, com foco em alta performance e facilidade de uso, que pudesse competir com soluções proprietárias.

Com isso, o MySQL tornou-se muito popular no ambiente web, sendo utilizado em conjunto com linguagens como PHP e Perl, especialmente no contexto do modelo LAMP (Linux, Apache, MySQL e PHP).

Em bancos de dados relacionais, um atributo representa uma coluna de uma tabela. Ele define um tipo de dado que será armazenado, como números, textos ou datas, e está ligado a uma característica da entidade que a tabela representa.

Já a chave primária (ou primary key) é um ou mais atributos que identificam unicamente cada registro de uma tabela. Ela não pode conter valores nulos nem duplicados, sendo essencial para manter a integridade dos dados.

Para criar uma tabela no MySQL, usamos o comando CREATE TABLE, seguido do nome da tabela e da definição de seus atributos e tipos. Também é possível definir a chave primária durante essa criação.

Exemplo de criação de uma tabela Aluno: 

                                                    CREATE TABLE Aluno (
                                                        matricula INT PRIMARY KEY,
                                                        nome VARCHAR(100),
                                                        idade INT,
                                                        curso VARCHAR(50)
                                                    );

Caso seja necessário excluir completamente uma tabela do banco de dados, utiliza-se o comando DROP TABLE, que remove a estrutura e todos os dados armazenados nela.

Exemplo de exclusão: 

                                                    DROP TABLE Aluno;

À medida que um sistema evolui, é comum que suas necessidades mudem. Por exemplo, pode surgir a necessidade de armazenar novas informações sobre os registros ou corrigir erros de modelagem feitos inicialmente. Nesses casos, não é necessário apagar e recriar a tabela — usamos o comando ALTER TABLE para alterar a estrutura de uma tabela existente.

Com o ALTER TABLE, podemos realizar diversas operações, como:

  • Adicionar colunas
  • Remover colunas
  • Modificar o tipo de dado de uma coluna
  • Renomear colunas
  • Alterar restrições, como chaves primárias ou únicas

Veja um exemplo onde precisamos adicionar uma nova informação sobre os alunos: o e-mail. 

ALTER TABLE Aluno
ADD COLUMN email VARCHAR(100);

Com esse comando, a tabela Aluno agora terá uma nova coluna chamada email, capaz de armazenar até 100 caracteres. Essa coluna será inicialmente nula para todos os registros já existentes, a não ser que você defina um valor padrão com DEFAULT.

Também é possível adicionar mais de uma coluna ao mesmo tempo: 

ALTER TABLE Aluno
ADD COLUMN telefone VARCHAR(20),
ADD COLUMN data_nascimento DATE;

Aqui, estamos adicionando duas colunas: uma para o número de telefone e outra para a data de nascimento dos alunos.

Suponha agora que uma informação deixou de ser necessária. Por exemplo, a coluna idade pode ser eliminada, pois a data de nascimento já fornece essa informação de forma mais precisa.

Para remover uma coluna, usamos: 

ALTER TABLE Aluno
DROP COLUMN idade;

Esse comando remove definitivamente a coluna idade da tabela. É importante ter certeza dessa operação, pois os dados armazenados nela serão perdidos.

Em outros casos, talvez você precise alterar o tipo de dado de uma coluna, por exemplo, se quiser aceitar nomes maiores ou números com casas decimais. 

ALTER TABLE Aluno
MODIFY COLUMN nome VARCHAR(150);

Aqui, estamos aumentando a capacidade da coluna nome de 100 para 150 caracteres. Essa mudança é útil quando se percebe que os dados reais são maiores do que o previsto inicialmente.

Outro exemplo seria transformar um número inteiro em decimal: 

ALTER TABLE Aluno
MODIFY COLUMN nota_final DECIMAL(5,2);

Agora, a coluna nota_final poderá armazenar números com até 3 dígitos antes da vírgula e 2 após, como 9.75 ou 100.00.

Em alguns SGBDs, como o MySQL, também é possível renomear colunas. Isso é útil quando o nome de uma coluna não representa mais corretamente a informação armazenada. 

ALTER TABLE Aluno
CHANGE COLUMN curso curso_atual VARCHAR(50);

Nesse exemplo, a coluna curso foi renomeada para curso_atual e manteve o mesmo tipo de dado.

O comando ALTER TABLE é uma ferramenta poderosa para manter seu banco de dados atualizado e consistente com as regras de negócio. Porém, como ele altera a estrutura das tabelas, é sempre recomendado fazer um backup antes de aplicar mudanças em bases de dados em produção.

Resumo das operações com ALTER TABLE:

  • ADD COLUMN – Adiciona uma nova coluna
  • DROP COLUMN – Remove uma coluna existente
  • MODIFY COLUMN – Altera o tipo ou restrição de uma coluna
  • CHANGE COLUMN – Renomeia e modifica uma coluna (MySQL)
