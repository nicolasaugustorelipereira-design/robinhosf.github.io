Exclusão de Dados (DELETE)

O comando DELETE é utilizado para remover registros específicos de uma tabela. Essa remoção é permanente e, por isso, deve ser utilizada com cuidado.

A estrutura básica do comando é: 

	DELETE FROM NomeDaTabela WHERE condição;

Exemplo prático: excluindo um aluno com matrícula 1001. 

	DELETE FROM Aluno WHERE matricula = 1001;

Nesse exemplo, o aluno com matrícula 1001 será permanentemente removido da tabela Aluno, caso exista.

Atenção: omitir a cláusula WHERE resultará na exclusão de todos os dados da tabela!

Exclusão de Tabelas (DROP TABLE)

O comando DROP TABLE é utilizado para excluir completamente uma tabela do banco de dados, incluindo todos os seus dados e estrutura.

Estrutura do comando: 

	DROP TABLE NomeDaTabela;

Exemplo prático: excluindo a tabela Aluno. 

	DROP TABLE Aluno;

Após a execução, a tabela Aluno deixará de existir no banco de dados, juntamente com todos os dados que ela continha.

Chaves Estrangeiras (FOREIGN KEY)

As chaves estrangeiras são usadas para criar relacionamentos entre tabelas. Elas garantem que os dados relacionados estejam consistentes entre si.

Uma chave estrangeira é uma coluna (ou conjunto de colunas) que aponta para a chave primária de outra tabela.

Exemplo de criação de uma tabela com chave estrangeira: 

	CREATE TABLE Matricula (
		id INT AUTO_INCREMENT PRIMARY KEY,
		aluno_id INT,
		curso_id INT,
		FOREIGN KEY (aluno_id) REFERENCES Aluno(matricula),
		FOREIGN KEY (curso_id) REFERENCES Curso(id)
	);

Neste exemplo, os campos aluno_id e curso_id são chaves estrangeiras que estabelecem relacionamentos com as tabelas Aluno e Curso, respectivamente.

  • A ordem das colunas deve corresponder à ordem dos valores informados.
  • Os valores devem respeitar os tipos de dados definidos na criação da tabela.
  • Se a tabela possuir uma restrição adicional, ela deve ser declarada corretamente.
